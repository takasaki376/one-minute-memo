2. ルーティングまわりの準備

/history

/themes

3. レイアウト&UI設計
レイアウトの方向性を決める

中央寄せのシンプルな縦並びレイアウトにする

余白・最大幅（例：max-w-xl）などのイメージを決める

Tailwind を使う場合のクラス構成をざっくり決める

タイトルのフォントサイズ

ボタンの大きさ・間隔

将来的に「簡易統計（総セッション数など）」を置くスペースを想定しておく（最初は空でOK）

4. コンポーネント利用の方針
共通 UI コンポーネント（Button, Card など）を使うかどうか決める

使う場合は src/components/ui/Button.tsx などを前提に、トップ画面での利用イメージを整理

5. 初期化処理（テーマ初期投入）との連携
「初期テーマ投入処理 (initBuiltinThemesIfNeeded)」をどこで呼ぶか決める

トップ画面のクライアントコンポーネント側で呼ぶか

もしくは別の初期化用コンポーネントに任せるか

トップ画面で呼ぶ場合のタスクとして

クライアントコンポーネント化（"use client"）の要否を検討

useEffect で一度だけ初期化処理を呼ぶ設計にする

6. 文言・コピーの作成
アプリ名・サブタイトルの文言を決める

例：one-minute-memo / 「1分で思考を書き出すメモアプリ」

各ボタンのラベル文言を決める

例：

「セッションを開始」

「履歴を見る」

「テーマを管理」

短い説明文を入れる場合は、1〜2文でまとめる

7. 表示確認・微調整
yarn dev で起動し、/ にアクセスして見た目を確認

スマホ幅（DevTools）とPC幅でレイアウトが崩れないかざっくり確認

/session

/history

/themes
