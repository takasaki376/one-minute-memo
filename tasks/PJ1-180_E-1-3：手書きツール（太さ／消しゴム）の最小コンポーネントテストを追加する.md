## 概要

手書きツール強化（E-1-1：ペン太さ / E-1-2：消しゴム）について、
**最低限のコンポーネントテスト**を追加し、主要な回帰を自動で検知できる状態を作る。

ピクセルレベルの描画検証は行わず、
「UIの切替」「描画設定が切り替わること（コンテキスト設定）」に絞って担保する。

---

## 背景 / 目的

- Canvas は UI変更やリファクタで壊れやすい
- 手動確認（E-1-4）だけだと回帰が見逃されやすい
- 自動テストで「致命的なポイントだけ」固定し、安心して改修できる状態にする

---

## 対象

- コンポーネント:
  - `src/components/session/HandwritingCanvas.tsx`
  - もしくはツールUIが SessionPage 側にある場合は `/session` のテストでも可
- テスト環境:
  - Vitest + Testing Library（既存構成に準拠）

---

## スコープ（やること）

### 1) ペン太さ UI の切替テスト

- 初期状態でデフォルト太さ（M）が選択されていること
- S/M/L をクリックすると選択状態（active）が切り替わること
  - class / aria 属性 / data-state などで判定

### 2) 描画コンテキスト設定の切替テスト（太さ）

- Canvas の `getContext('2d')` をモックし、
  `lineWidth` が選択した太さに応じて設定されることを確認する
- どのタイミングで設定するか（描画開始時 or 切替時）に合わせて
  テストの期待値を調整する

### 3) 消しゴム UI の切替テスト

- 初期状態で tool = pen であること
- 消しゴムを選ぶと tool が eraser になり、active 表示が切り替わること
- ペンに戻せること

### 4) 描画コンテキスト設定の切替テスト（消しゴム）

- `globalCompositeOperation` が
  - pen: `source-over`
  - eraser: `destination-out`
    に切り替わることを確認する
- 可能なら、消しゴム時の lineWidth（固定 or ペン太さ連動）も確認する

## 実装メモ

- jsdom では Canvas の 2D context が実装されていないことが多いため、
  `HTMLCanvasElement.prototype.getContext` をモックするのが現実的
- 例（概念）:
  - `const ctx = { lineWidth: 0, globalCompositeOperation: 'source-over', ... }`
  - `getContext` が ctx を返すようにする
  - クリック → pointerdown などのタイミングで ctx の値を検証
- UI状態判定は `aria-pressed` や `data-state="on"` のように
  安定したフックを用意するとテストが壊れにくい（必要なら追加）
