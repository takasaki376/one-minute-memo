「共通レイアウト作成」のタスクは、ざっくり言うと

全ページで共通になる “枠組み” を app/layout.tsx で決める

作業です。
ここでは、そのタスクの中身をやることベースで分解します。

🎯 このタスクのゴール
どのページに行っても

同じヘッダー

同じ背景・余白・フォント
が効いている状態にする。

すべてのページが layout.tsx を経由して表示される。

✅ 共通レイアウト作成タスクの詳細ステップ
1. レイアウト方針を決める
やること（頭の中 or メモでOK）：

ページ幅：max-w-3xl or max-w-4xl 程度の中央揃え？

余白：上下 py-6 / py-10 くらいにする？

背景色：淡いグレー or 白ベース？

ヘッダーに置くもの：

アプリ名（one-minute-memo）

ナビゲーションリンク：

Home

セッション開始

履歴

テーマ管理

※ ここで「こんな感じにしたい」というざっくりレイアウトを決めておきます。

2. app/layout.tsx を編集する
やること：

app/layout.tsx を開く

HTML の <html> / <body> 構造を定義

共通ヘッダー・mainコンテナを追加

フォントや背景色など基本のスタイルを指定（Tailwind 前提）

例（あくまでイメージ）：



// app/layout.tsx
import type { Metadata } from 'next';
import './globals.css';

export const metadata: Metadata = {
  title: 'one-minute-memo',
  description: '1分で思考を書き出すメモアプリ',
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="ja">
      <body className="min-h-screen bg-slate-50 text-slate-900">
        <div className="min-h-screen flex flex-col">
          {/* 共通ヘッダー */}
          <header className="border-b bg-white">
            <div className="max-w-4xl mx-auto px-4 py-3 flex items-center justify-between">
              <div className="font-semibold">
                one-minute-memo
              </div>
              <nav className="flex gap-4 text-sm">
                {/* ここは後で <Link> に差し替える */}
                <a href="/">Home</a>
                <a href="/session">セッション</a>
                <a href="/history">履歴</a>
                <a href="/themes">テーマ管理</a>
              </nav>
            </div>
          </header>

          {/* 各ページの中身 */}
          <main className="flex-1">
            <div className="max-w-4xl mx-auto px-4 py-6">
              {children}
            </div>
          </main>

          {/* フッター（必要なら） */}
          <footer className="border-t bg-white">
            <div className="max-w-4xl mx-auto px-4 py-3 text-xs text-slate-500">
              © {new Date().getFullYear()} one-minute-memo
            </div>
          </footer>
        </div>
      </body>
    </html>
  );
}
※ ナビ部分はあとで next/link に置き換えるタスクを別で切ってもOKです。

3. 共通コンポーネントに切り出す（余裕があれば）
レイアウトタスク内に含めてもいいし、別タスクにしてもOK。

やること：

components/ui/Header.tsx を作成

layout.tsx からヘッダー部分を切り出す

将来、モバイル対応やロゴ差し替えがしやすくなる

4. globals.css との整合を確認
やること：

app/globals.css に Tailwind の @tailwind base; などがあることを確認

ベースフォント（日本語・英語どちらも見やすいもの）を指定したければここで

例（任意）：



/* app/globals.css のどこかに */
body {
  font-family: system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI',
    sans-serif;
}
5. 動作確認
タスク完了条件：

yarn dev して以下を目視確認

/ /session /history /themes にアクセスしたとき、

同じヘッダー・余白・背景になっているか

main コンテナ内の幅や余白が想定通りか

ページを増やしても 何もしなくても同じレイアウトが適用される 状態になっている

まとめ：このタスクでやることリスト
チェックリストにすると：

レイアウトの方針を決める（幅・余白・ヘッダー内容）

app/layout.tsx に共通レイアウト（ヘッダー＋コンテナ＋フッター）を実装

必要ならヘッダーを components/ui/Header.tsx に切り出す

globals.css とフォント／背景設定を確認

全ページで共通レイアウトが効いていることを確認

このうち 2 の layout.tsx 実装をもっと具体的にコードレベルで細かく見たいとか、
ヘッダーの Link 周りだけ別タスクとして設計したいなどあれば、そこをピンポイントで掘ります。
