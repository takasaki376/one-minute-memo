## 概要

タブレット向け分割レイアウト（Story A-1）および
回帰防止対応（Story B-1）について、
**手動で確認すべき項目をチェックリストとして整理・共有**する。

自動テストではカバーしきれない
「実操作感・端末特有の挙動」を担保することが目的。

---

## 背景 / 目的

- Canvas 操作・タッチ入力・画面回転などは自動テストでの網羅が難しい
- タブレット対応は UI/UX 影響が大きく、目視確認が不可欠
- 確認観点を明文化し、属人化せずに再現性のある確認を行えるようにする

---

## 対象画面

- `/`（トップ画面）
- `/session`（セッション画面）
- `/session/complete`（セッション完了画面）
- `/history`（履歴一覧）
- `/history/[id]`（履歴詳細）

---

## 作成物

- 手動チェックリスト（Markdown）
  - 置き場所例:
    - `docs/manual-test/tablet-session.md`
    - または PR コメントテンプレート
    - または Jira コメントに直接貼り付け

---

## チェックリスト内容（案）

### 1. 表示・レイアウト

- [ ] スマホサイズ（md未満）で既存の縦積みレイアウトになっている
- [ ] タブレットサイズ（md以上）で2ペインレイアウトになる
- [ ] Canvas が主領域として大きく表示される
- [ ] TextEditor が補助領域として表示される
- [ ] PCサイズでもレイアウトが崩れない

### 2. 画面回転・リサイズ

- [ ] 縦向き → 横向きで Canvas サイズが追従する
- [ ] 横向き → 縦向きで描画が欠けない
- [ ] 回転後も描画位置がズレない

### 3. 手書き入力（Canvas）

- [ ] 描画できる
- [ ] 描画中に画面スクロールが発生しない
- [ ] Canvas 外では通常通りスクロールできる
- [ ] 手書き入力が途中で中断されない

### 4. テキスト入力（TextEditor）

- [ ] 入力・編集ができる
- [ ] スクロールが可能
- [ ] Canvas 操作と干渉しない
- [ ] キーボード表示時にレイアウトが破綻しない

### 5. セッション進行

- [ ] セッション開始できる
- [ ] 各テーマで入力 → 保存 → 次テーマに進める
- [ ] タイマー満了時に自動保存される
- [ ] 最終テーマ後に `/session/complete` に遷移する

### 6. 保存・履歴

- [ ] セッション完了後、履歴一覧に表示される
- [ ] 履歴詳細でテキスト/手書きが正しく表示される

---

## 非スコープ（やらないこと）

- 実端末の網羅的テスト（全機種）
- パフォーマンス測定
- 新機能追加
- UX改善の提案

---

## 完了条件（Definition of Done）

- チェックリストが Markdown で作成されている
- チーム内で共有されている（PR / Jira / docs）
- タブレット対応PRで、このチェックリストに基づく確認結果が残せる

---

## 運用メモ

- 初回は iPad 相当サイズでの確認を想定
- 実端末が無い場合は Chrome DevTools のデバイスモードでも可
- 不具合が見つかった場合は、新規チケットとして切り出す
