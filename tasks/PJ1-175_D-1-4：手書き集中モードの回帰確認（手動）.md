## 概要

手書き集中モード（D-1-1〜D-1-3）導入後に、
**既存機能および分割レイアウト（split）に回帰が発生していないこと**を
手動チェックで確認する。

本チケットは「品質担保・安全な出荷」が目的であり、新規機能追加は行わない。

---

## 背景 / 目的

- 集中モードはレイアウト分岐・オーバーレイ・状態保持など、
  回帰を生みやすい要素が多い
- 特に以下が壊れやすい
  - セッション進行（保存/次テーマ/完了）
  - Canvas描画の保持（再マウント）
  - TextEditor の入力保持（開閉）
  - タイマーの継続
- 手動確認観点を固定し、属人化せずに確認できる状態にする

---

## 確認対象画面

- `/session`（セッション画面）
  - split（分割）モード
  - handwritingFocus（集中）モード
- `/session/complete`
- `/history`
- `/history/[id]`

---

## チェックリスト（手動）

### 1) モード切替の基本

- [ ] md以上でモード切替UIが表示される
- [ ] split → handwritingFocus に切り替えられる
- [ ] handwritingFocus → split に戻せる
- [ ] 切替時に画面が破綻しない（レイアウト崩れがない）

### 2) 状態保持（最重要）

- [ ] モード切替しても Canvas の描画が消えない
- [ ] モード切替しても TextEditor の入力が消えない
- [ ] モード切替しても タイマー状態（残秒/進行）が維持される
- [ ] 切替によって保存・進行の state が壊れない

### 3) TextEditor オーバーレイ（集中モード）

- [ ] 集中モード中に TextEditor を開ける
- [ ] TextEditor に入力して閉じても、再度開くと入力が残っている
- [ ] オーバーレイ表示中、Canvas が誤反応しない
- [ ] オーバーレイの閉じる導線（×/背景/ESC等）が機能する

### 4) セッション進行（保存/次テーマ/完了）

- [ ] 中間テーマで入力→保存→次テーマが正しく進む
- [ ] タイマー満了時に自動保存される（仕様通り）
- [ ] 手動操作とタイマー満了が競合しても二重保存/二重遷移しない
- [ ] 最終テーマで saveMemo → completeSession → /session/complete へ遷移する

### 5) 画面回転・リサイズ（タブレット想定）

- [ ] 縦→横、横→縦で Canvas が追従し表示が欠けない
- [ ] 回転後も描画位置がズレない
- [ ] TextEditor の開閉が回転後も問題なく動作する

### 6) 回帰確認（既存機能）

- [ ] md未満（スマホ）で集中モード導線が出ない（または影響がない）
- [ ] split（分割）モードの2ペイン表示が従来通り
- [ ] 履歴一覧/詳細でデータが正しく表示される
- [ ] 既存の保存データが破損していない

## 非スコープ（やらないこと）

- 自動テスト追加（別チケットで対応）
- UX改善提案・追加実装
- パフォーマンス測定
- 全機種網羅（iPad以外の端末ごとの検証）

---

## 完了条件（Definition of Done）

- 上記チェックリストを一通り確認し、致命的な問題がない
- 問題が見つかった場合は「再現手順＋期待/実際」を記録し、別チケット化されている
- 確認結果が PR コメントまたは Jira コメントに残っている

---

## 記録フォーマット（推奨）

- 確認端末/環境:
  - 例: Chrome DevTools iPad, 実機 iPadOS Safari 等
- 結果:
  - OK / NG
  - NGの場合は再現手順とスクリーンショット（可能なら）
