# 設定画面 開発タスク一覧

## 1. 設計・前提確認

1. **設定仕様の確定**

   - テーマ件数：`min=1 max=100`（整数・クランプ）
   - 入力時間：`min=1 max=3600`（整数・クランプ）
   - 適用タイミング：次回セッション開始から
   - 初期値：テーマ 10 / 時間 60

2. **データモデル確定（Settings）**
   - `theme_count: number`
   - `time_limit: string`（既存に合わせる）  
     または `time_limit_seconds: number`（内部表現改善）
   - `id = "default"` の単一レコード運用方針

---

## 2. IndexedDB / データアクセス層

3. **openDB のスキーマ更新**

   - `settings` ストア追加（`keyPath: id`）
   - 既存 DB_VERSION の扱い整理  
     （開発初期なら DB 削除 / そうでなければ version up）

4. **settingsRepo 実装**

   - `getSettings(): Promise<Settings>`
   - `updateSettings(patch): Promise<Settings>`（部分更新）
   - `resetSettings(): Promise<Settings>`（初期値に戻す）

5. **デフォルト値フォールバック実装**
   - 取得失敗や未作成時に  
     `{ theme_count: 10, time_limit: "60" }` を返す
   - 初回起動時に default レコードを作るか、  
     読み込み時に仮想的に扱うかを決定して実装

---

## 3. hooks / 状態管理

6. **useSettings フック実装 or 更新**

   - `settings, isLoading, error, updateSettings`
   - 内部で `settingsRepo` を呼ぶ
   - `updateSettings` を optimistic にするか（MVP）判断

7. **エラー / ローディング状態の共通化**
   - `Loader` 表示条件の整理
   - `error` 表示 UI（メッセージ + 戻る導線）

---

## 4. UI コンポーネント（入力）

8. **InputTargetCount コンポーネント実装**

   - `localCount` の同期（`value` 変更で追従）
   - `onBlur` 時の差分更新のみ `onUpdate`
   - `NumberInput` 設定（min / max / 整数 / strict clamp）

9. **InputTargetTime コンポーネント実装**

   - `localTime` の同期（`value` 変更で追従）
   - `onBlur` 時の差分更新のみ `onUpdate`
   - `NumberInput` 設定（min / max / 整数 / strict clamp）

10. **単位表示（件・秒）とヘルプテキスト**
    - ラベル・補足文言の整備
    - `htmlFor` / `id` の関連付け

---

## 5. SettingPage（画面）実装

11. **`src/app/(authenticated)/setting/page.tsx` 骨組み作成**

    - ヘッダー（タイトル、閉じる `MdOutlineClose` → `/` 遷移）
    - 2 カラム / カード UI（Tailwind）

12. **`useSettings` 連携（読み込み表示）**

    - `isLoading` → `Loader`
    - `error` → エラーメッセージ
    - `settings` → 入力に反映

13. **`onBlur` 更新の wiring**

    - `InputTargetCount` → `updateSettings({ theme_count })`
    - `InputTargetTime` → `updateSettings({ time_limit })`

14. **推定所要時間の表示（新規）**

    - `theme_count * Number(time_limit)` の計算
    - 秒・分表記（例：600 秒（10 分））
    - local state に追従させる（推奨）

15. **適用ルール注記の表示**

    - 「設定は次回セッション開始から適用されます…」

16. **「初期値に戻す」機能**

    - UI ボタン追加
    - 挙動をどちらかに統一
      - A: 即時 `updateSettings({ theme_count: 10, time_limit: "60" })`
      - B: 値のみ戻して blur で保存（MVP は A 推奨）

17. **フォーム submit 抑止**
    - `onSubmit` で `preventDefault()`

---

## 6. ルーティング / ナビゲーション

18. **Setting への導線実装**

    - トップ or メニューから `/setting` へ遷移

19. **privacy ページの存在確認**
    - `src/app/(authenticated)/setting/privacy/page.tsx` の確認

---

## 7. セッション開始画面への反映（重要）

20. **/session 開始時に Settings を適用**

    - `createSession` 前に `getSettings()` を取得
    - テーマ抽選件数に `theme_count` を反映
    - `useCountdown.initialSeconds` に `time_limit` を反映

21. **適用タイミングの担保**
    - 進行中セッションに影響しないことを確認
    - セッション開始時に settings をスナップショット保存（推奨）

---

## 8. テスト

22. **settingsRepo 単体テスト**

    - 初回はデフォルトが返る
    - update で部分更新できる
    - reset で初期値に戻る

23. **useSettings 単体テスト**

    - ローディング → 読み込み → 更新の遷移

24. **SettingPage コンポーネントテスト**

    - 初期値表示
    - blur で update 呼び出し
    - 初期値に戻す動作
    - 推定所要時間の表示

25. **/session 連携テスト（回帰）**
    - 設定変更後にセッション開始し、
      テーマ数 / 時間が反映される

---

## 9. 動作確認（手動）

26. **設定画面の最終チェック**

    - クランプが効く
    - 推定所要時間が追従
    - 注記が表示される
    - 閉じるで `/` に戻れる

27. **反映確認**
    - 設定変更 → 新規セッションで反映
    - 進行中セッションは影響なし
