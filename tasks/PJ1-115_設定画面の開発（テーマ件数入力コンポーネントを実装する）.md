## Task 3-1: テーマ件数入力コンポーネント（InputTargetCount）を実装する（詳細）

### Sub-task 3-1-1: コンポーネント責務・API を確定する

- 目的: テーマ件数（theme_count）を編集できる入力 UI を提供する
- Props を確定する（推奨）
  - value: number
  - onUpdate: (count: number) => Promise<void> | void
  - min?: number（デフォルト 1）
  - max?: number（デフォルト 100）
  - id?: string（label 紐付け用）
  - disabled?: boolean（読み込み中/更新中に対応する場合）
  - description?: string（ヘルプテキスト）
- エラー表示はページ側（SettingPage）で行うため、Input 単体には error 表示責務を持たせない（MVP）

### Sub-task 3-1-2: UI 仕様を確定する

- ラベル: テーマ件数
- 単位: 件（入力の右または補助テキスト）
- ヘルプテキスト: テーマの出題数を設定
- NumberInput 制約
  - min=1
  - max=100
  - allowDecimal=false
  - clampBehavior="strict"
  - step=1

### Sub-task 3-1-3: ローカル state 同期の実装方針を確定する

- 内部 state: localCount を保持する
- value が外から変わったら localCount を更新する（useEffect）
- 入力中は localCount を更新し続ける

### Sub-task 3-1-4: onChange ハンドリングを実装する

- NumberInput の onChange で localCount を更新する
- 受け取る値が number | string | null の可能性を考慮して安全に変換する
- null の場合は直前値を維持するか、min に寄せるか方針を決める（推奨: 直前値維持）

### Sub-task 3-1-5: onBlur 更新（差分のみ）を実装する

- onBlur 時に「value と localCount が異なる」場合のみ onUpdate を呼ぶ
- 呼び出し値は min/max でクランプした値を渡す
- 失敗時の挙動（MVP）
  - onUpdate が throw/reject してもフォームは維持（localCount はそのまま）
  - エラー表示は親（SettingPage）に任せるため、Input は例外を握りつぶさない（親で catch）
  - ただし「連続で onUpdate が走る」ことは避けたいので、失敗後の value 反映ポリシーを決める
    - 推奨: 親が settings を更新できなかった場合 value は変わらない → 次回 blur でも差分あり → 再試行できる

### Sub-task 3-1-6: a11y（label 紐付け）を実装する

- label の htmlFor と NumberInput の id を一致させる
- aria-describedby（ヘルプテキスト）を紐付けできるなら対応する

### Sub-task 3-1-7: Storybook/開発用確認（任意）

- 単体でレンダリングし、min/max/clamp と blur 更新が期待通りか確認する

## Task 3-2: 入力時間コンポーネント（InputTargetTime）を実装する（詳細）

### Sub-task 3-2-1: コンポーネント責務・API を確定する

- 目的: 入力時間（time_limit）を編集できる入力 UI を提供する
- Props を確定する（既存設計を踏襲）
  - value: string（既存）※内部で Number に変換して表示
  - onUpdate: (time: string) => Promise<void> | void
  - min?: number（デフォルト 1）
  - max?: number（デフォルト 3600）
  - id?: string
  - disabled?: boolean
  - description?: string
- エラー表示はページ側（SettingPage）で行う（MVP）

### Sub-task 3-2-2: UI 仕様を確定する

- ラベル: 入力する時間
- 単位: 秒
- ヘルプテキスト: テーマごとの制限時間を設定
- NumberInput 制約
  - min=1
  - max=3600
  - allowDecimal=false
  - clampBehavior="strict"
  - step=1

### Sub-task 3-2-3: ローカル state 同期の実装方針を確定する

- 内部 state: localTime（string）を保持する
- value が外から変わったら localTime を更新する（useEffect）
- 画面表示は Number(localTime) を利用（NaN 対策あり）

### Sub-task 3-2-4: onChange ハンドリングを実装する

- NumberInput の onChange で localTime を更新する
- 入力値が null の場合の扱いを決める
  - 推奨: localTime を変更しない（直前値維持）
- `localTime` は string 化して保持（既存設計維持）

### Sub-task 3-2-5: onBlur 更新（差分のみ）を実装する

- onBlur 時に value と localTime が異なる場合のみ onUpdate を呼ぶ
- 呼び出し値は min/max でクランプした結果を string 化して渡す
- 失敗時の挙動（MVP）
  - フォームは維持（localTime はそのまま）
  - 親でエラー表示、入力は継続可
  - 親の settings が更新されない限り value は据え置き → 次の blur で再試行できる

### Sub-task 3-2-6: a11y（label 紐付け）を実装する

- label htmlFor / input id の一致
- aria-describedby を必要に応じて追加

### Sub-task 3-2-7: プリセット（任意・将来）

- MVP では不要だが、将来拡張として 30/60/90/120 秒ボタンを追加できるように設計余地を残す
- 追加する場合の props も検討（presets?: number[]）

## Task 3-3: 入力制約・アクセシビリティ・エラー表示方針を整備する（詳細）

### Sub-task 3-3-1: clamp 仕様の統一

- theme_count: min=1 max=100 strict clamp
- time_limit: min=1 max=3600 strict clamp
- strict clamp の意味（範囲外は入力できない/戻る）を確認し、UX を許容できるか判断する

### Sub-task 3-3-2: 入力の型変換・NaN 対策を統一

- NumberInput の onChange が返す型を想定し、共通ユーティリティを作るか決める
  - 例: `toIntOrNull(value)` / `clampInt(n, min, max)`
- local state が NaN を抱えないようにガードする

### Sub-task 3-3-3: 「更新失敗してもフォーム維持」の実装責務を明確化する

- Input コンポーネントは例外を握りつぶさず親に伝播（推奨）
- SettingPage 側が catch して上部にエラー表示しつつフォームは維持する
- 画面上部のエラー表示 UI の仕様を簡単に決める
  - 例: 赤背景のテキストボックス / `text-red-600` の 1 行メッセージ

### Sub-task 3-3-4: disabled 条件（任意）を設計する

- isLoading 中は入力を disabled にするかどうか
- 更新中（isUpdating）を導入するなら、その間だけ disabled も検討
- MVP では「ローディング中は Loader 表示」で入力自体を出さないため、disabled は必須ではない

### Sub-task 3-3-5: コンポーネント単体テストを実装する（推奨）

- InputTargetCount
  - value 表示が正しい
  - change → blur で onUpdate が呼ばれる（差分がある時のみ）
  - change しても blur しなければ onUpdate が呼ばれない
- InputTargetTime
  - value 表示が正しい
  - change → blur で onUpdate が呼ばれる（差分がある時のみ）
  - min/max クランプ結果が渡される
