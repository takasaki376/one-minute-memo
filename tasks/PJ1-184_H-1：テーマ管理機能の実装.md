# Epic: テーマ管理機能の実装

ユーザーがテーマを一覧で確認・検索・有効/無効化・追加・編集できるようにし、
あわせて `builtinThemes` を初期データとして DB に登録する仕組みを実装する。

---

## Story 1: 初期テーマデータ（builtinThemes）をDBへ登録できるようにする

### Task 1-1: builtinThemes を初期投入用データとして整理する

- `builtinThemes` のデータ構造を ThemeRecord と整合させる
- 必要な項目を確定する
  - title
  - category
  - isActive
  - source
- 初期投入時に必要な付加情報
  - id
  - createdAt
  - updatedAt
    をどこで補完するか決める

### Task 1-2: themes ストアの前提を整理する

- themes ストアに保存するレコード構造を確定する
- `source: 'builtin' | 'user'` を保持できるようにする
- 一意性の扱い（title重複許容/非許容）を決める

### Task 1-3: builtinThemes 初期投入処理を実装する

- DB に初期テーマが未投入の場合のみ登録する処理を追加する
- 約500件の builtinThemes を一括投入できるようにする
- 重複投入を防止する

### Task 1-4: 初期投入済み判定を実装する

- すでに投入済みであれば再投入しない
- 判定方法を決める
  - themes 件数
  - builtin source の有無
  - 専用フラグ
- 開発時に再投入しやすい運用方法も整理する

### Task 1-5: アプリ起動時の初期投入導線を組み込む

- 初回利用時に builtinThemes が自動で DB に投入されるようにする
- どの画面/タイミングで実行するかを決めて組み込む
  - トップ画面初期化
  - テーマ管理画面表示時
  - 共通初期化処理

### Task 1-6: 初期投入機能の動作確認を行う

- 初回起動で約500件が投入される
- 再読み込みしても重複登録されない
- 既存テーマが壊れない
