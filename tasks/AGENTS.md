# タスク記載フォーマット（共通ルール）

- 冒頭に 1〜2 行のひとことコメントでモチベーションや背景を書き、その後に水平線（`---` または `----`）を入れて本文を開始する。
- 「## ?? タスク名」セクションを置き、イタリックでタスク名を示しつつ対象パスやルートを `{{src/...}}` `{{/route}}` などのダブル波括弧で明示する。セクション間は水平線で区切る。
- 本文は番号付きセクション（`## 1.` 〜）で大枠を分け、それぞれの中でやることを番号付きリストで列挙し、必要に応じて `#*` / `#**` などのプレフィックスで入れ子のサブタスクを記載する。
- ファイル名・関数名・プロパティ名・ルートはすべて `{{...}}` で囲んで強調する。コード例を示す場合は Markdown のコードブロックで囲み、内部でも `{{}}` を用いて文脈を明示する。
- セクションタイトルは役割ごと（画面仕様の整理、Repository 準備、データ取得ロジック、UI レイアウト、ナビゲーション、ローディング・エラー、動作確認など）に分け、各セクション内は「決めること」「やること」をタスク形式で書く。
- 最終セクションに「## ? 完了条件（DoD）」を置き、到達すべき状態を箇条書きでまとめる（必要に応じて `**` などで 2 階層の表記を使う）。
